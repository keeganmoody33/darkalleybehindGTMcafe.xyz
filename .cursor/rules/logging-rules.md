# Logging Rules

Every work session must:
1. Read progress.txt first.
2. Summarize current state before changing code.
3. Log what was completed.
4. Log what is currently in progress.
5. Log what is next.
6. Log known bugs/blockers.
7. Log files touched.
8. Log verification performed.

Every architecture decision must be written to docs/ADR.md.

Every inspiration item must include:
- source / reference name
- link or screenshot path
- what to borrow
- what not to copy
- which product area it influences
- status: observed / adopted / rejected

Every new lesson must include:
- problem
- root cause
- prevention rule
- affected files / systems
